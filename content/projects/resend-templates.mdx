## The problem

Every new project meant rewriting the same emails: account confirmation, password reset, purchase receipt. Each one took a few hours to get right — typed, responsive, dark mode.

I decided to extract these into a reusable library.

## The solution

A set of ready-to-use templates with:

- **Shared design system**: consistent color tokens, typography, and spacing across all templates
- **Full TypeScript types**: each template has a typed interface for its props
- **Local preview**: via `react-email` you see the result before sending
- **Dark mode**: native support in clients that handle it

## Available templates

- `WelcomeEmail` — post-signup welcome
- `ResetPasswordEmail` — password recovery link
- `OrderConfirmationEmail` — purchase receipt with line items and total
- `InvoiceEmail` — invoice with tax breakdown
- `MagicLinkEmail` — passwordless authentication

## Technical decisions

### React Email over raw HTML

Inline-HTML email templates are a maintenance nightmare. React Email brings components, composition, and typing — the output is still plain HTML, but the source is readable.

## Result

Reduced email setup time in new projects from ~4 hours to ~20 minutes.
