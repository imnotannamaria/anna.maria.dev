TypeScript without `strict: true` is like wearing a seatbelt without buckling it.

## What strict mode enables

When you enable `strict` in `tsconfig.json`, you activate a set of checks TypeScript skips by default:

```typescript
// Without strict: this compiles fine
function greet(name: string | null) {
  return name.toUpperCase() // 💥 potential runtime error
}

// With strict: the compiler stops you
function greet(name: string | null) {
  return name?.toUpperCase() ?? "Hello!"
}
```

## Why the initial friction is worth it

The learning curve exists, but the ROI is high. Strict mode catches entire categories of bugs at compile time — null dereferences, implicit any, missing return types — that would otherwise surface in production.

Once you internalize the patterns, writing strict TypeScript is faster than debugging runtime errors.
