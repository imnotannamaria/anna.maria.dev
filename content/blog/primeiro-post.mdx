TypeScript sem `strict: true` é como usar cinto de segurança sem apertar a fivela.

## O que o strict mode ativa

Quando você habilita `strict` no `tsconfig.json`, você está ativando um conjunto de checagens que o TypeScript não faz por padrão...

```typescript
// Sem strict: isso passa
function greet(name: string | null) {
  return name.toUpperCase() // 💥 runtime error potencial
}

// Com strict: o compilador te para
function greet(name: string | null) {
  return name?.toUpperCase() ?? "Olá!"
}
```
